Shared vs. Solo Accounts

SaaSquatch supports companies with accounts that are shared between users (B2B apps) and solo accounts (B2C apps) out of the box.

What is the difference between Shared and Solo accounts?

Shared accounts have many users using the same billing and solo accounts don't.

Shared Account Solo Account
Multiple users share the same billing information Every user has their own billing

account_id vs. payment_provider_id

account_id is your internal identifier, whuke payment_provider_id

account_id payment_provider_id
Used to groups users into a single account Used to lookup an account in the payment system. E.g. Stripe customer id or Recurly account id

Free or trial users aren't in my payment system

If you're using shared accounts, then you can just set payment_provider_id: null for those accounts.
