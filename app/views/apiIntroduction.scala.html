API Reference

The Referral SaaSquatch API is organized around REST. Our API is designed to have predictable, resource-oriented URLs and to use HTTP response codes to indicate API errors. We use built-in HTTP features, like HTTP authentication and HTTP verbs, which can be understood by off-the-shelf HTTP clients. JSON will be returned in all responses from the API, including errors.

To make the Referral SaaSquatch API as explorable as possible, accounts have test-mode API keys as well as live-mode API keys. These keys can be active at the same time. Data created with test-mode credentials will use your payment system's respective test mode and will never cost anyone money.

API Endpoint https://app.referralsaasquatch.com/

Summary of Resource URL Patterns

  • /api/v1/{TENANT_ALIAS}/coupon
  • /api/v1/{TENANT_ALIAS}/coupon/{CODE}
  • /api/v1/{TENANT_ALIAS}/account/{ACCOUNT_ID}
  • /api/v1/{TENANT_ALIAS}/subscription
  • /api/v1/{TENANT_ALIAS}/subscription/{URL}
