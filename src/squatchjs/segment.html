How do I use the Segment integration?

Use identify from Segment.io instead of squatch.js init to identify the currently logged in user and set up SaaSquatch.

Some of the parameters are different. For example, to add an image for a user changed from user_image to avatar.

Squatch.js Segment / analytics.js
Use squatch.js init Use analytics.js identify 
_sqh.push(["init", {
    tenant_alias: "test_aaaexampleaaa",
    user_id: "u1234",
    account_id: "a5678",
    email: "joe.tester@example.com",
    user_image: "http://example.com/avatar/205e460b479e2e",
    first_name: "Joe",
    last_name: "Tester",
    fb_share_image: "http://www.example.com/logo.png",
    checksum: "arbPDAcedO38Qw/qdJLCqd2tlRQ="
}]);
 
analytics.initialize({
 "SaaSquatch": { tenantAlias: "test_aaaexampleaaa"
}});

analytics.identify( "u1234", {
    accountId: "a5678",
    email: "joe.tester@example.com",
    avatar: "http://example.com/avatar/205e460b479e2e",
    firstName: "Joe",
    lastName: "Tester",
    referralImage: "http://www.example.com/logo.png",
    checksum: "arbPDAcedO38Qw/qdJLCqd2tlRQ="
});

How do I use Signed Requests with Segment.io?

You can still use Signed Requests with Segment, keeping in mind a few things:

  • Include tenant_alias in the checksum
  • Omit other analytics.js traits from the checksum, such as Google Analytics ids
  • Ordering is based upon squatch.js init parameters names -- not the analytics.js parameter names! For example, avatar should still be sorted with "u" for user_image.

What features are supported with Segment?

Most of the features of SaaSquatch are released immediately for Segment. You can always use squatch.js directly to open or close the referral widget. There will always be some things that will require you to use squatch.js directly.

Can I used the REST API with Segment?

No, you cannot use the SaaSquatch REST API on the server side with Segment. At this time Segment is only an abstractions of the squatch.js javascript library.
